Class Diagram:
A Class Diagram is the most important UML diagram because it represents the static structure of an object-oriented system.

Components of a Class Diagram:
1. Class: A class represents a blueprint for creating objects.
+-------------------+
| Student           |
+-------------------+
| id: int           |
| name: string      |
+-------------------+
| study()           |
| writeExam()       |
+-------------------+

Class Name: Located in the first compartment.
    Usually singular noun
    Starts with uppercase
    Center aligned
    Can be abstract or concrete

Example:
    Student
    Employee
    Order
    Customer
    Product

2. Attributes (Fields / Properties):
    Located in the second compartment.
    Represent the state of the class. [name, data type, default value]
    Can be public, private, or protected.
    Example:
        id: int
        name: string
        age: int

+-------------------+
| Student           |
+-------------------+
| age:int = 18      |
+-------------------+


3. Operations (Methods): Located in third compartment.

+-------------------+
| Student           |
+-------------------+
| id:int            |
+-------------------+
| study()           |
| writeExam()       |
+-------------------+

Syntax: visibility methodName(parameters):returnType

+calculateSalary():float
+login(username,password):bool
